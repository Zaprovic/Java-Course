Encapsulation

Hiding functionalities of a class to the user can result in a better implemented code, especially to the user, because if we let the user change literally everything in a code, that can turn out messing everything pretty bad, leading to exceptions and errors.

The main way to tackel this things is by using getter and setter methods. Consider the following code:

public class Enemy{

private int health;

//setter method
public void setHealth(int x){
x = health;

}

//getter method
public void getHealth(){
System.out.println("Health: " + health);
}

public void talk(){
System.out.println("I am going to kill you");
   }

}

public class Vampire extends Enemy{
@Override
public void talk(){
System.out.println("I am going to suck your blood");
   }
}

There are two methods implemented to set the health of an enemy equal to a number, and to retrieve that number as well. A good way of using this in a driver class would be as follows:

Good practice:

class Main{
public static void main(String[] args){
Enemy vampire1 = new Vampire();
}

vampire1.setHealth(75);
vampire1.getHealth();
}

This way is so much better and cleaner, because in less lines of codes and a more dynamic implementation we can set the health of an enemy and retrieve it back.

Bad practice

class Main(){

public static void main(String[] args){
Enemy vampire1 = new Vampire();

vampire1.health = 50;

System.out.println(vampire1.health);
   }
}
